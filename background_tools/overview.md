# Background Tools

Before diving into AI-specific workflows, it helps to have a solid foundation in a few general-purpose tools that are widely used in research and software development. The pages in this chapter cover:

- **Git and GitHub** — version control and collaborative project management
- **Markdown** — a lightweight syntax for writing formatted documents (including this one)
- **Text Editors** — choosing and configuring an editor that supports your workflow
- **Unit Tests and Test-Driven Development** — writing automated tests to verify code correctness and using tests to guide development (covers Python, R, and Stata)

These tools are not AI-specific, but they come up constantly in AI-assisted research contexts. Many AI tools generate markdown output, operate inside editors, and are best used alongside a version-controlled project structure. Testing is especially relevant when working with AI coding assistants: writing tests first gives you a precise, machine-checkable specification that you can hand to the AI and use to verify its output.
